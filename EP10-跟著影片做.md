## 🎯 範例 1：純前端工具「課堂抽籤器」（5 分鐘）

**目標**：做一個學生座號隨機抽籤的網頁。重新整理就重置，不需要記住資料 = 範式 A。

### Step 1：在 Google 雲端硬碟開新資料夾

```
G:\我的雲端硬碟\classroom-picker\
```

### Step 2：Claude Code 桌面版「新增專案」→ 選這個資料夾

### Step 3：對 Claude 說（複製貼上）

```
請幫我做專案起手式，這個專案叫做 classroom-picker。
專案說明：課堂抽籤器（從學生座號隨機抽一位）。
```

→ Claude 自動建 `CLAUDE.md` / `.gitignore` / git init / GitHub 公開 repo + push。

### Step 4：寫網頁

```
請做一個單頁網頁 index.html，叫「課堂抽籤器」。
功能：
- 預設座號 1-32（要可調整範圍）
- 一個大大的「抽籤」按鈕
- 抽中的座號用超大字顯示
- 可重複抽（避免抽到同一個有個小提示）
風格：黑底白字、橘色強調、像粉筆寫在黑板上。
完成後請預覽給我看。
```

### Step 5：上線

```
都沒問題，請 push 到 GitHub 並開啟 Pages，告訴我網址。
```

→ 1 分鐘後拿到網址 `https://你的帳號.github.io/classroom-picker/`，可以掃 QR 投影。

**🎉 你做出了第一個工具！**

## 🎯 範例 2：學生提交 + 公開展示「即時文字雲」（30 分鐘）

**目標**：學生提交詞彙，大螢幕即時顯示文字雲。要記住資料 + 多人同時用 = 範式 B。

### Step 1：開新資料夾 + 起手式

```
G:\我的雲端硬碟\classroom-wordcloud\
```

對 Claude 說：「請幫我做專案起手式，叫 classroom-wordcloud，做課堂即時文字雲。」

### Step 2：建 Firebase 後端（複製貼上）

```
請用 Firebase 做這個專案的後端。

集合名稱：wordcloud_words
每筆資料的欄位：
- word（字串）：學生輸入的詞彙
- created_at（時間戳）：自動產生

請：
1. 用 Firebase MCP 建立新專案（或用我現有的）
2. 建 Firestore 資料庫，地點選 asia-east1
3. 寫 firestore.rules：wordcloud_words 集合公開可讀可寫，其他全部禁止
4. 部署規則
5. 接下來要做兩個 HTML 頁面
```

> ⏸ Claude 會在中途請你開 cmd 跑 `firebase login`。照做、回對話框繼續。

### Step 3：做兩個 HTML 頁面

```
請做兩個 HTML 頁面，都連到剛剛的 Firebase。

頁面一：submit.html（學生提交頁）
- 大大的輸入框 + 提交按鈕
- 提交後寫入 wordcloud_words 集合
- 學生送完顯示「謝謝你的回答」就好
- 風格：手機掃 QR 開、輸入要方便

頁面二：display.html（公開展示頁）
- 用 onSnapshot 即時顯示所有詞彙
- 出現次數越多字越大（文字雲視覺）
- 風格：投影到大螢幕的視覺效果，深色底
```

### Step 4：本機預覽 → 上線

```
請預覽兩個頁面給我看，沒問題就 push 到 GitHub 並開啟 Pages，告訴我網址。
```

### Step 5：上課現場使用

- 投影 `display.html` 到大螢幕
- 把 `submit.html` 的網址做成 QR Code 給學生掃（或對 Claude 說「請幫我做這個 URL 的 QR Code」）
- 學生用手機提交 → 大螢幕即時長出文字雲

### Step 6：老師端查資料（隨時對 Claude 講）

```
請查 wordcloud_words 集合，告訴我：
- 今天總共幾筆
- 最熱門的 5 個詞（按出現次數排序）
```

```
請刪除 wordcloud_words 集合裡 word 等於「測試」的所有資料。
```

**🎉 你做出了第一個有資料庫的工具！**
